# 矩阵乘法的梯度 — 对 B 求导

## 问题

对于 `Y = A @ B`，已知 `dL/dY`（上一层传来的梯度），求 `dL/dB`。

## 具体例子

A 是 (2, 3)，B 是 (3, 2)，Y 是 (2, 2)：

```
Y = [a11 a12 a13]   [b11 b12]
    [a21 a22 a23] @ [b21 b22]
                    [b31 b32]
```

## 推导过程

**第一步**：展开 Y 的每个元素

```
y11 = a11·b11 + a12·b21 + a13·b31
y12 = a11·b12 + a12·b22 + a13·b32
y21 = a21·b11 + a22·b21 + a23·b31
y22 = a21·b12 + a22·b22 + a23·b32
```

**第二步**：对 b11 求梯度（看哪些 y 用到了 b11）

只有 y11 和 y21 用到了 b11，用链式法则：
**dL/db11 = (dL/dy11)·(∂y11/∂b11) + (dL/dy21)·(∂y21/∂b11)**

```
dL/db11 = (dL/dy11)·a11 + (dL/dy21)·a21
```

同理对 b21、b31：

```
dL/db21 = (dL/dy11)·a12 + (dL/dy21)·a22
dL/db31 = (dL/dy11)·a13 + (dL/dy21)·a23
```

**第三步**：写成矩阵形式

```
[dL/db11  dL/db12]     [a11  a21]   [dL/dy11  dL/dy12]
[dL/db21  dL/db22]  =  [a12  a22] @ [dL/dy21  dL/dy22]
[dL/db31  dL/db32]     [a13  a23]
```

即：

```
dB = A.T @ dY
```


## 记忆方式

对 B 求梯度 → 把**另一个矩阵 A** 转置，从左边乘 dY。

# Q / K / V 的理解（2026-05-26）

## 1. Q、K、V 是什么

输入 X 经过三个不同的线性变换，分别得到 Q、K、V：

```
Q = X @ W_Q
K = X @ W_K
V = X @ W_V
```

其中 W_Q、W_K、W_V 是可学习的参数矩阵。它们来自同一个输入，但因为乘了不同的权重矩阵，所以各自侧重不同的信息。

直觉上的比喻：

- **Q（Query）**：当前 token 在"问"什么——"我需要找什么样的信息"
- **K（Key）**：每个 token 提供的"标签"——"我有什么样的信息"
- **V（Value）**：每个 token 实际的"内容"——"我的信息具体是什么"

## 2. 注意力是怎么算的

```
Attention(Q, K, V) = softmax(Q @ K.T / √d_k) @ V
```

**第一步**：`Q @ K.T` 算匹配度

- Q 的第 i 行和 K 的第 j 行做点积
- 点积越大 → 匹配度越高 → 第 i 个 token 越关注第 j 个 token

**第二步**：`softmax` 归一化成权重（和为 1 的概率分布）

**第三步**：`权重 @ V` 加权求和，得到融合了上下文信息的新表示

## 3. Q "表示需要什么信息" 是什么意思

> **曾经的问题**：Q 的每一行就是一个 token 的向量，为什么说它"表示需要什么信息"？它明明只是一堆数字。

Q 本身并没有显式地写着"我需要 X"。这个含义是**训练出来的**。

一开始 W_Q 是随机初始化的，Q 就是乱七八糟的向量，和 K 算出来的匹配度也是随机的，翻译结果当然是错的。

但训练过程中，梯度从 Loss 一路传回来，不断调整 W_Q 和 W_K：

```
W_Q ← W_Q - lr × ∂L/∂W_Q
W_K ← W_K - lr × ∂L/∂W_K
```

模型通过反复试错发现：当某些 Q 和 K 算出高分时，翻译结果更准确、Loss 更低。于是参数就往这个方向调整。

**所以 Q "表示需要什么" 不是人为设计的，而是模型自己学出来的。** 我们人类为了方便理解，才用"它在找什么信息"来解释。

## 4. 自注意力 vs 交叉注意力

### 自注意力（Q = K = V = 同一个输入）

- **Encoder 自注意力**：中文 token 之间互相看，理解句子内部结构。例如"学"和"生"组成一个词，它们的 Q 和 K 会算出较高的匹配度。
- **Decoder 自注意力**：英文 token 之间互相看（有因果 mask，不能看未来），理解"已经翻译了什么"。

### 交叉注意力（Q 和 K/V 来自不同地方）

- Q 来自英文（Decoder）："我现在要翻译第 2 个位置"
- K 来自中文（Encoder）："我"、"是"、"学"、"生" 各自的标签
- V 来自中文（Encoder）："我"、"是"、"学"、"生" 各自的内容

交叉注意力是翻译的核心：英文的 Q 去"查"中文的 K，找到最相关的中文字，然后把它的 V 取过来帮助翻译。

### 信息流向

| 场景             | Q 来源 | K 来源 | V 来源 |
| ---------------- | ------ | ------ | ------ |
| Encoder 自注意力 | 中文   | 中文   | 中文   |
| Decoder 自注意力 | 英文   | 英文   | 英文   |
| 交叉注意力       | 英文   | 中文   | 中文   |

信息是**单向流动**的：中文 → 英文。没有"英文 K 被中文 Q 查"这一步。

## 5. 梯度是怎么传回来的

Loss 从最终输出**一层一层传递**回来到每一层的参数：

```
Loss ← 交叉熵 ← 线性投影层(W_out)
     ← Decoder 交叉注意力
     ← Decoder 自注意力
     ← Encoder 自注意力
     ← W_Q, W_K, W_V
```

每一层都拿到上一层传来的梯度，用链式法则算出自己的参数该怎么调，再继续往回传。最终传到 W_Q、W_K 时，梯度告诉它们："你们上次算的匹配模式导致翻译错了，应该这样调整。"

这就是为什么 Q 和 K 能学到有意义的表示——**梯度从最终 Loss 一路传回来，指导每一层的参数往正确的方向更新。**

## 6. 总结

1. Q、K、V 是同一个输入的三种不同线性投影，分别代表"找什么"、"有什么"、"给什么"
2. `Q @ K.T` 算匹配度，softmax 归一化，再乘 V 得到加权结果
3. Q "表示需要什么信息" 不是人为设计的，是训练出来的
4. 自注意力（Q=K=V）找句子内部关系，交叉注意力（Q 和 K/V 不同）找跨语言对应
5. W_Q、W_K 通过梯度下降学习，梯度从 Loss 经链式法则逐层传回
